# PWS_Blockchain_Final
PWS Blockchain derde deelvraag (final).
- Dowload node.js --> https://nodejs.org/en/download (Na de setup, controleer of nodeJS werkt: open cmd en type "node -v". Mijn huidige versie is "v20.16.0", dus node.js met v20 of hoger moet dit project wel aan).

Om het project te laten  runnen (onderste uitleg id voor windows, kan ook op macOS, maar dan met macOS commands natuurlijk):
- download dit repo (project).
- open cmd > cd repo > type "npm install" om de benodigde modules (node_modules) te downloaden. 
- om het project te laten runnen, open meerdere terminals en cd naar het project in alle geopende terminals, als eerste laat "npm run node_1" (node 1) runnen bij één terminal (in het begin moet deze node runnen, want het gossip gaat via deze node, dit is de vaste node). Daarna kunt u de andere nodes laten runnen bij de andere gopende terminals, zoals "npm run node_2" (node 2) etc. Per terminal 1 "npm run node" command. Kijk package.json>"scripts" voor node_1 node_2, node_3 en node_4 commands. Als default zijn er 4 nodes, als u bijvoorbeeld 5 nodes wilt, dan moet u nog een command toeveoegen in package.json>"scripts" --> als "node_5": "node --watch dev/networkNode.js 3005", voor de zesde node, geldt node_6 en 3006 enzovoort.

- Als u niet meerdere terminals tegelijkertijd kunt laten runnen: type in de terminal: "Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass". After this command, temporary, you can run commands in multiple cli's. (Dit is voor windows, voor macOS denk ik niet dat u deze fout zal krijgen). 
- De hoofdpagina van een node is: "http://localhost:${PORT}". ${PORT} is dan 3001, 3002, 3003 of 3004. Ligt eraan welke node u heeft opgestart naast node_1 (node_1 moet altijd opgestart zijn).
- Heel zelden krijg ik de 'melding' van de --watch flag van nodeJS, dan wordt het project automatisch opgestart (alsof iets in networkNode.js is gewijzigd), dan wordt de sessie ook opnieuw opgestart en wordt er dan een nieuwe blockchain gemaakt en de gebruikers zijn ook vernieuwd. Bij de voorafgaande projecten is dit een grotere probleem omdat ik daar naar de hele directory keek, en soms kwamen hierdoor fouten bijvoorbeeld wanneer je een andere gebruiker wilde toevoegen. Maar in dit project is er alleen maar "node --watch dev/networkNode.js 3001", dus we kijken alleen naar de newworkNode.js of die gewijzigd is. Dus hierdoor bijna geen meldingen van de watch flag van nodeJS.
